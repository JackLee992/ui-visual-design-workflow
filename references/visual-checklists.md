# Visual Checklists

在 `SKILL.md` 主流程之后使用这些检查表。

## 通用构图

- 先写一句话：“观众应该先看到 ___，再理解 ___，最后做 ___ 或感到 ___。”
- 首屏/封面只保留一个主焦点。
- 用尺寸、明度、颜色、间距、位置和细节密度建立层级。
- 栅格对齐更稳重可信；非对称更有能量；居中对称只在仪式感或图标感成立时使用。
- 相关元素靠近；无关元素用间距、分隔线或容器分开。
- 眯眼看或缩到缩略图检查图底关系。
- 删除不支持信息、节奏或品牌的装饰。

## UI 界面

- 从用户任务开始，不从视觉风格开始。
- 先排内容优先级：主操作、次操作、状态、导航、帮助/错误。
- 使用语义色彩角色：主操作、surface、正文、弱文本、成功、警告、危险、焦点。
- 包含必要状态：默认、hover/focus、active、选中、禁用、加载、空状态、错误、成功。
- 标签短而明确，避免只有 placeholder 没有 label 的表单。
- 尽量测试 200% zoom 或大字号模式。
- 检查触控目标、安全区、响应式行为和文字不重叠。

## 海报与社交图

- 让主题/活动/利益点在 1-2 秒内可读。
- 只使用一个图像钩子或字体钩子；不要让标题、照片、徽章和装饰同时抢注意力。
- 文本角色保持简单：标题、辅助说明、详情、CTA/日期。
- 重要文字远离裁切边缘和印刷出血区。
- 印刷品确认 bleed、色彩模式、图片分辨率和导出格式。
- 社交图先检查最小信息流缩略图，再定稿。

## 小红书图文

- 交付前核对最新平台规格。实用基线：除非内容明显适合方图，否则封面和轮播优先围绕 3:4 竖版裁切设计。
- 封面标题短而具体。优先使用结果、对比、错误、清单、数字、身份线索。
- 封面文字必须在信息流缩略图可读。背景复杂时使用高对比、阴影、描边或半透明文字底板。
- 轮播图片保持同一比例，降低裁切风险。
- 组织节奏：封面承诺、问题/场景、步骤或证据、案例、总结、收藏/分享提示。
- 避免二维码、联系方式、外站水印、误导性 before/after、关键词堆砌、重复低原创图片。
- 优先使用真实场景证明、清晰特写、标注说明和 before/after，不要只做泛化装饰图。

## 游戏 CG 与 Key Art

- 用一句话定义游戏承诺：类型、幻想、冲突、角色或核心机制。
- 三档尺寸都要读得懂：商店缩略图、普通卡片、全图。
- 角色或主体要有清晰剪影和明度分组。
- 背景每个元素都要服务焦点、故事、尺度或世界氛围。
- logo/title 背后要有干净明度区域，不能压在高频细节上。
- 商店 capsule 除非平台明确允许，不要添加额外营销文字。
- HUD/UI 要确认文本、小地图、瞄准、血量、警告和图标在亮/暗/动态背景上可见。

## AI 出图 Prompt Pack

如果用户明确要最终图片或图片变体，prompt pack 才是出图前的准备步骤；准备完成后转入 `$imagegen`，用内置 `image_gen` 生成或编辑位图。否则 prompt pack 本身就是可交付产物，不自动出图。

使用这个结构：

```text
Subject: ...
Intent: ...
Composition: ...
Camera/lens/framing: ...
Lighting: ...
Palette/value: ...
Style/medium: ...
Materials/details: ...
Mood: ...
Platform crop: ...
Text policy: no text in image unless explicitly required
Negative constraints: ...
Variations: ...
```

需要多方向探索时，给三种方向：稳妥/品牌向、强视觉/编辑向、实验向。用评分表比较后再选最终方向；轻量请求可以只给一个推荐方向。
