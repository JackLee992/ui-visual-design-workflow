# UI Visual Design Workflow

## 核心原则

把视觉设计当成“引导注意力”，不是单纯装饰。每次设计都要说清楚：目标人群、核心信息、第一视觉焦点、希望用户做什么或感受到什么、平台限制、验收标准。

## 快速分流

先判断用户要的是哪一类产物：

- **UI 界面/产品设计**：App、网页、后台、组件、Figma screen。
- **海报/社交图**：活动海报、广告、banner、封面、缩略图、公告图。
- **小红书图文**：封面、轮播、生活方式攻略、购物种草、教程、对比笔记。
- **游戏 CG/key art**：splash art、Steam capsule、loading art、角色或场景主视觉。
- **AI 出图指导**：prompt、moodboard、风格探索、位图生成。
- **视觉诊断/改稿**：审查已有设计图或图片。

## 执行模式

默认使用轻量模式，除非用户明确要求升级：

- **轻量模式（默认）**：不联网、不调用 imagegen；只读取本技能和必要检查表，交付设计简报、设计策略、规格、prompt 或 QA。
- **联网核验**：仅当用户要求最新平台规则、尺寸、政策、审核限制、店铺素材规则，或明确说“查一下/联网确认”时使用。优先官方平台文档、设计系统文档、可访问性标准、成熟 UX 研究；通常 1-3 个高可信来源足够。
- **成品出图**：仅当用户明确要生成或编辑最终位图成品、图片变体、封面图、海报图、CG、key art、产品图或 UI mockup 位图时，同时使用 `$imagegen` 和内置 `image_gen`。如果用户说“只要提示词/方案/规范/先别出图”，禁止调用 imagegen。

## 工作流程

1. **明确简报**
   用一段话写清：目标、受众、平台、画布尺寸/比例、必须出现的文字、品牌/风格限制、已有素材、期限、成功指标。缺信息但风险不高时，可以自行假设并标注。

2. **参考扫描**
   轻量模式下可用已有知识、用户素材或本地资料提炼参考属性。只有进入联网核验模式，或用户明确要求找参考时，才联网找 3-7 个参考；不照抄作品，只提取可复用属性：构图、层级、色彩情绪、字体语气、图片处理、内容节奏、平台惯例。不要模仿在世艺术家或单一可识别作品。

3. **设计策略**
   先定注意力路径：
   - 一个主焦点；
   - 两到三个信息层级；
   - 清楚的图底关系；
   - 用邻近、容器或线条表达分组；
   - 用栅格和对齐建立秩序；
   - 用留白提高信噪比。

4. **视觉系统**
   明确定义：
   - **色彩**：语义角色、对比度目标、明暗背景适配、必要时说明文化含义。
   - **字体**：1-2 组字体、3-5 个文本角色、最小可读字号、行高、中文可读性。
   - **图像**：主体、裁切、光线、景深、质感、写实/插画程度、素材来源。
   - **版式**：栅格、边距、安全区、阅读顺序、响应式或裁切行为。
   - **交互状态**：UI 需包含默认、hover/focus、选中、禁用、加载、空状态、错误、成功。

5. **媒介规则**
   请求涉及 UI、海报、小红书、游戏 key art 或 AI prompt 时，读取 `references/visual-checklists.md` 的对应检查表。

6. **输出设计规范**
   给新手可执行的规格：
   - 设计简报；
   - 画布和导出要求；
   - 版式/构图方案；
   - 色彩、字体、图像规则；
   - 必备文案；
   - 如需出图，提供 prompt pack；
   - QA 检查表。

7. **关联 imagegen 出图**
   仅在成品出图模式下使用 `$imagegen`，并通过内置 `image_gen` 工具生成或编辑图片。不要因为用户提到 UI、海报、prompt、moodboard 或视觉风格就自动出图；这些请求默认先交付方案、规格、prompt pack 或 QA。出图前先用本技能产出清晰美术指导稿；出图后按本技能的 QA 检查表复查焦点、层级、文字、裁切和平台适配。

8. **自检改稿**
   对每项打 0-2 分：焦点清晰、层级清晰、可读性、可访问性、平台适配、品牌适配、可交付性。任何 0 分项必须先修改，再声称完成。

## 质量门槛

- 文案在目标展示尺寸下必须可读，尤其是移动端缩略图。
- 不要只靠颜色传达含义；同时使用形状、标签、图标、纹理或位置。
- UI 和重文字图以 WCAG 对比度为基线：普通文本 4.5:1，大号文本和重要非文本 UI 线索 3:1。
- AI 图像 prompt 要像美术指导稿：主体、动作、构图、镜头、光线、色板、风格、情绪、材质、平台裁切、负面约束。
- 小红书等高变化平台，只有在联网核验模式下才写死最新比例、上传数量、违规项、披露要求；否则标注为实用基线或待核验。
- 游戏商店素材在需要导出或发布前核对对应平台规则；部分平台限制宣传语、奖项、评分和额外文字。

## 参考资料使用

- 需要来源依据、平台核验或引用时，且任务进入联网核验模式，读取 `references/source-map.md`。
- 需要按媒介执行或诊断时，读取 `references/visual-checklists.md`。
- 需要游戏图片类型生成示例、成品出图 prompt 模板或测试样张时，读取 `references/game-image-generation-samples.md`。
- 用户明确要生成或编辑最终位图成品时，必须同时使用 `$imagegen`，并优先调用内置 `image_gen` 工具。
- 用户要创建或更新 Figma 文件时，先使用对应 Figma 技能，再调用 Figma 工具。
